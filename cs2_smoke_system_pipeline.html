01 · 시스템 레이어 아키텍처

CS2 연막은 단순 스프라이트가 아닌 4개 레이어의 혼합 시스템입니다. 각 레이어가 어떤 역할을 맡는지 먼저 구조를 파악해야 UE5.7 구현 전략이 결정됩니다.

Layer A
Fluid Grid Sim
유체 그리드 시뮬레이션 — 볼륨 내부를 3D 셀 그리드로 표현. 각 셀에는 밀도(density) 와 속도(velocity) 값이 있으며, 매 틱마다 advection으로 전파됩니다. 총알/폭발 시 해당 셀에 임펄스를 더해 밀도를 분산시킵니다.
Houdini Pyro DOPs
UE5 Niagara Grid3D
Layer B
Volume Raymarch
볼륨 레이마칭 렌더링 — 밀도 필드(density field)를 볼륨 텍스처로 읽어 카메라 레이가 볼륨을 통과하며 누적 불투명도를 계산. 연막의 두께감과 내부 라이팅이 여기서 결정됩니다.
UE5 Heterogeneous Volumes
Sparse Volume Textures (SVT)
Layer C
Screen Space
스크린 스페이스 보정 — Custom Depth와 씬 깊이를 비교해 연막 경계의 Soft Fade, 캐릭터와 연막의 교차 처리, 화면 공간 조명 근사치를 보완합니다.
Custom Depth Stencil
Post Process Material
Layer D
Gameplay Sync
게임플레이 연동 — 서버에서 그리드 상태(요약본)를 전송하고, 클라에서 시각적 보간 처리. GAS로 그레네이드 투척 → 연막 스폰 → 상태 변이 흐름을 관리합니다.
GAS GameplayAbility
Blueprint Event Interface
구현 범위 설정: 4개 레이어를 동시에 처음부터 구현하면 복잡도가 폭발합니다. UE5.7 학습 목적이라면 Layer A (Grid3D) + Layer B (Heterogeneous Volume) 조합부터 시작하고, Layer C/D는 이후 단계에서 붙이는 방식이 현실적입니다.

Houdini DOP Pyro Sparse 셋업 핵심
Container Type: Sparse 필수 — Dense는 메모리가 고정이지만 Sparse는 연막이 있는 셀만 계산합니다.

DOP Network 구성:
  • pyro_import SOP → 소스 지오메트리
  • Smoke Object → 초기 밀도 세팅
  • PyroSolver → advect + diffuse
  • Wind Force / Drag Force → 외력
Houdini SOP Velocity → Direction Map 변환
시뮬 결과의 vel 속성(float3 벡터)을 텍스처 공간으로 인코딩하는 과정:

1. VDB Analysis → velocity 필드 추출
2. Attribute Remap → [-max, +max] → [0, 1]
3. Labs Bake Texture → R=velX, G=velY, B=velZ
4. 출력: 16-bit EXR (정밀도 유지) 권장
# Python (Houdini 21.0) — VDB 시퀀스를 배치로 UE5 임포트용 .vdb로 변환 import hou import os node = hou.node('/obj/pyro_sim/OUT_vdb') output_dir = '/export/smoke_vdb/' frame_range = (1, 120) # 120프레임 베이킹 for frame in range(*frame_range): hou.setFrame(frame) # density + velocity 채널 분리 export geo = node.geometry() vdb_density = geo.primGroups()['density'] vdb_vel = geo.primGroups()['vel'] filename = os.path.join(output_dir, f'smoke_{frame:04d}.vdb') node.parm('file').set(filename) node.cook(True) print(f"Exported {frame_range[1] - frame_range[0]} frames to {output_dir}")
Labs FlowMap 노드 활용: Houdini Labs의 Labs FlowMap SOP은 메시 위 커브 흐름 방향을 자동으로 UV 방향 맵으로 베이킹합니다. 연막 바닥면 흐름 애니메이션 제작 시 UV 방향 맵과 FlowMap을 조합하면 더 풍부한 결과를 얻을 수 있습니다.
03 · UE5.7 — 볼륨 렌더링 구현 (Heterogeneous Volumes + SVT)
UE5.7 RENDER Pre-baked VDB → Sparse Volume Texture → 볼륨 레이마칭
UE5.7 Sparse Volume Textures (SVT)
UE5.1+부터 .vdb 파일을 직접 SVT 에셋으로 임포트할 수 있습니다.

임포트 경로:
Content Browser → Import → .vdb

생성되는 에셋 타입: USparseVolumeTexture

채널 구성:
  • Attribute A → density (float)
  • Attribute B → temperature / vel (float3)
UE5.7 Heterogeneous Volume 컴포넌트
SVT를 씬에 배치하는 컴포넌트.
Actor → Add Component → Heterogeneous Volume

주요 설정:
  • Sparse Volume Texture — 임포트한 SVT 에셋 연결
  • Max Step Count — 레이마칭 스텝 수 (품질 vs 성능)
  • Extinction Scale — 밀도 감쇠 강도
  • Albedo — 연막 산란 색상
  • Frame Rate / Sequence — VDB 시퀀스 애니메이션

UE5.7 Heterogeneous Volumes + Lumen: Lumen은 Heterogeneous Volume의 조명 산란을 자동으로 처리합니다. 별도 라이팅 코드 없이도 씬의 다이렉셔널 라이트 / 스카이 라이트가 연막 내부를 자연스럽게 투과합니다. 다만 Heterogeneous Volume은 현재 동적 그림자(Dynamic Shadow)를 자체 투영하지 않으므로 볼륨 그림자가 필요하면 Volumetric Fog 레이어를 보조로 추가해야 합니다.
04 · Niagara Grid3D — 런타임 반응형 유체 시뮬
NIAGARA GPU에서 실시간으로 유체 그리드를 시뮬레이션하는 경로

Pre-baked SVT는 아름답지만 총알/폭발에 반응하지 못합니다. CS2처럼 연막이 실시간으로 변형되려면 Niagara의 Grid3D Collection으로 GPU에서 직접 Advection 루프를 돌려야 합니다.

Grid3D 핵심 개념
Grid3D Collection은 3D 배열을 GPU 버퍼에 유지하는 Niagara 데이터 인터페이스입니다.

연막에 필요한 채널:
  • Density — float, 셀 당 연막 농도
  • Velocity — float3, 셀 당 이동 방향/속도
  • Pressure — float (divergence-free 보정용)
Niagara 시스템 구성
Stage 1: Advect Velocity
vel = SampleGrid(pos - vel*dt)

Stage 2: Apply Forces
vel += Impulse * dt

Stage 3: Advect Density
density = SampleGrid(pos - vel*dt)

Stage 4: Render Proxy
density → Sprite/Volume 출력
Grid3D 해상도 가이드
Grid3D 해상도는 성능과 품질의 직접 트레이드오프입니다.

32³저사양 / 거친 시뮬
64³권장 (게임 런타임)
128³고품질 (PS5급 타겟)

CS2는 서버 시뮬을 32³ ~ 64³ 수준으로 추정됩니다.
// Niagara HLSL — Grid3D Advection Stage (Simulation Stage) // Grid3DCollection Interface에서 호출하는 커스텀 HLSL 블록 float3 pos = CellIndex.xyz; // 현재 셀 좌표 (0~resolution) // 1. 현재 셀의 속도 읽기 float3 vel = GridRead_Velocity(pos); // 2. 역방향으로 추적 (Semi-Lagrangian advection) float3 backPos = pos - vel * DeltaTime; // 3. 이전 위치에서 밀도 샘플링 (쌍선형 보간) float newDensity = GridSample_Density(backPos); // 4. 자연 소산 (연막이 시간이 지나면 옅어짐) newDensity *= (1.0 - DissipationRate * DeltaTime); // 5. 결과 저장 GridWrite_Density(pos, newDensity); // === 외부 임펄스 적용 (총알 / 폭발) === float3 worldPos = GridToWorld(pos); float dist = length(worldPos - ImpulseOrigin); float strength = exp(-dist / ImpulseRadius) * ImpulseStrength; float3 impulseDir = normalize(worldPos - ImpulseOrigin); vel += impulseDir * strength; GridWrite_Velocity(pos, vel);
Semi-Lagrangian Advection 이란: 각 셀이 "나는 어디서 왔는가?"를 역방향으로 추적하는 기법입니다. 현재 위치 - 속도×시간 = 이전 위치 → 그 위치의 값을 현재 셀로 복사합니다. 단순하지만 안정적이라 게임 실시간 유체에 표준적으로 사용됩니다. (CS2도 이와 유사한 방식으로 추정)

07 · 스크린 스페이스 보정 — Custom Depth + Post Process
UE5.7 Soft Edge (연막 경계 부드럽게)
연막이 바닥/벽과 만나는 경계를 자연스럽게 처리하려면:

1. 연막 메시 Actor → Render CustomDepth Pass: ON
2. Post Process Material에서:
SceneDepth - CustomDepth → Fade Mask
Fade 거리: 약 30~80 UU

3. 이 Fade Mask를 볼륨 Extinction에 곱하면
연막이 지오메트리 표면에서 자연스럽게 사라집니다.
UE5.7 연막 내부 Occlusion 처리
연막 내부에서 먼 쪽 물체를 가리는 처리:

방법 A: 레이마칭 자체 감쇠
Heterogeneous Volume의 Extinction 값이 충분히 높으면 자동 처리됩니다.

방법 B: Depth Prepass
연막 Actor에 Translucency Sort 를 활용해 깊이 정렬이 올바르게 이뤄지도록 합니다.

방법 C: Lumen 신뢰
UE5.7 Lumen은 볼륨 자체 조명/차폐를 상당 부분 처리해줍니다.
